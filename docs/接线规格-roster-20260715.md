## 结论

侦察结论:①核实 5108 行的 srcset 是【标题 KV】(artURL('title')/'titleSmall'),不是名册封面——Codex 没有预留任何名册封面代码,封面属全新增。②代码里【不存在 TITLES 常量】;称号的唯一事实源是 metaTitle()(index.html:3769-3772,门槛 lv>=20 断云弟子 / >=15 首徒 / >=10 高徒 / >=5 学徒 / else 见习生),与 manifest 的 rankOrder 五档一一对应。metaDefault() 里的 titles:[] 是【死字段】,全文件从未被读或写(grep 只在 3744 出现一次)——不要给它加语义。③levelOf(xp) 返回 {lv,cur,need,pct} 对象(3760-3765),4036 行注释还留着"当数字用"的历史 bug 记录,所有纹章档位一律走 levelOf(x).lv。④18 个成就 ID 逐一核对 manifest.achievementMap:a_first/a_clear/a_nodeath→victory,a_deep20/a_slay1/a_slay10→shadow,a_bounty10/a_rich→bounty,a_star3/a_star10/a_lv10→growth,a_elem4/a_bridge→tactics,a_runs10/a_runs50→devotion,a_seen20/a_seenAll/a_mvp5→collection,共 18 个,与 ACHIEVEMENTS(3777-3796)【完全吻合,零缺口零多余】。⑤最关键的技术坑:12 枚纹章全是纯 currentColor SVG(无任何显式颜色),用 <img src> 加载时 currentColor 在 SVG 独立文档里会解析成【黑色】,在暗金底上等于隐形——所以必须走 CSS mask-image + background:currentColor;而 mask 资源 404 时 mask-image 计算为 none 会露出【纯色实心方块】,故用一次性 Image() 探测(rosterArtProbe)成功才加 .rart 类点亮,探测失败/file:// 缺图则纹章 display:none,名册完全退回现状。⑥不 bump ART_VER(新目录新文件名无缓存冲突),也【不动 ART 映射对象】(避免与其他域的并行编辑抢同一个锚点),名册美术用自带的 ROSTER_DIR/rosterArt() 小helper,复用 ART_VER 做 cache-busting 并保持 file:// 不带 ?v 的既有约定。⑦全部只读 _meta,零 S 写入,零存档边界改动。

## 风险与注意

- mask 兼容:iOS Safari 需 -webkit-mask-* 前缀,规格里已前缀/标准双写;若某端两者都不支持,纹章将不显示(display 仍由 .rart 控制,不会露色块),名册退回纯文字——可接受。
- rosterArtProbe 只探一枚 roster-rank-novice.svg 作代表:若单枚成就 SVG 被误删而其余在,该枚会露出 currentColor 实心方块。整包同目录同批交付,风险低;若要绝对稳,可改成逐图探测但会多 12 个请求。
- 封面 20:7 很扁:手机竖屏(modal 内容宽约 330px)封面仅约 115px 高,右侧安全区靠 clamp() 缩字撑住(称号约 10px / Lv 约 19px / 纹章 30px)。真机 430×932 必须实测一眼,若挤,把 .rc-lv 的 clamp 下限降到 17px 即可,不要改 aspect-ratio。
- sizes="calc(100vw - 100px)" 精确对应 veil padding 20×2 + modal-box padding 30×2;若将来 #modal-box 的 padding 改了,这个 sizes 要同步,否则 DPR 选源会偏一档。
- ACH_ART 是硬编码映射:将来 ACHIEVEMENTS 新增第 19 个成就时,不加映射会静默落到 victory 兜底——第 2 条自检(files.size===7 + 长度 18)会在那时直接报红,这是有意的护栏。
- 未 bump ART_VER、未改 ART 对象:如果主 Claude 在别的域里 bump 了 ART_VER,第 3 条自检里的 ART_VER==='20260713f' 断言需要同步改,否则会红。

## 缺图/失败回退

三级回退,任何一级都不白屏不报错:①封面 WebP 加载失败 → onerror 移除 <img> 并给容器加 .rc-noart,容器塌回 aspect-ratio:auto、去边框去底色、.rc-in 变 static 居中 → 视觉等价于改造前的居中「称号 + 名册 Lv.N」标题块;②纹章资源探测失败(离线 file:// 缺目录、404) → 不加 .rart,.r-em/.ach-em 保持 display:none → 称号文字与 18 个成就 chip 文字完全维持现状(✔ 与成就名本来就是真实 DOM 文本,不依赖图);③mask 属性不被支持 → 同样只是纹章不显示,不会露出实心色块。全程零 S 写入、零 localStorage 写入、零 ART_VER/UI_ICON_VER 改动,存档与图标自检不受影响。
